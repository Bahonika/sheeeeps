# ТЗ Stage 2: локальный мультиплеер

## Цель
Дать возможность 2–4 друзьям играть вместе в одной сессии по локальной сети. Каждый управляет своей собакой, стадо общее, цель общая — загнать всех овец в вольер. Кооператив, без PvP.

Распространение максимально простое: один и тот же exe у всех, никакого выделенного сервера, никаких аккаунтов, никакой инфраструктуры в интернете. Один игрок жмёт «Создать комнату» и становится хостом, остальные подключаются к нему по локальной сети. Аудитория — небольшая группа знакомых, масштабирование не нужно.

## Модель сети
**Хост-авторитарная.** Вся симуляция (овцы, физика, стадный инстинкт, условие победы) исполняется только на хосте — это тот же игровой код, что в одиночной игре, никакого отдельного серверного билда. Клиенты отправляют хосту только ввод, получают обратно состояние мира и рисуют его.

- Клиент → хост: цель движения своей собаки (точка клика), событие гавка. Больше ничего.
- Хост → клиенты: снапшоты состояния 15–20 раз в секунду: позиции и состояния всех овец, позиции всех собак, счётчик «в вольере», кулдауны гавков, флаг победы.
- Клиенты интерполируют позиции между снапшотами, чтобы движение выглядело плавным при 15–20 Гц обновлений. Экстраполяция/предсказание не нужны — для кооператива по LAN задержки в единицы миллисекунд, интерполяции достаточно.
- При 300+ овцах снапшот целиком может быть тяжёлым — упаковывать позиции компактно (например, координаты как int16, состояние овцы одним байтом) и/или слать дельты относительно предыдущего снапшота. Начать с простого полного снапшота в бинарном виде, дельты добавлять только если замеры покажут проблему.

## Транспорт
WebSocket поверх TCP внутри локальной сети. Хост поднимает встроенный WebSocket-сервер на фиксированном порту (например 7777) прямо в игровом процессе средствами стандартной библиотеки Dart — отдельный процесс не нужен. Для LAN надёжности TCP хватает, UDP и борьба с потерями пакетов — лишнее усложнение на этом этапе.

## Обнаружение комнаты
Два способа подключения, оба должны работать:

1. **Автопоиск по LAN.** Хост раз в секунду шлёт UDP-broadcast с именем комнаты, версией игры и своим адресом/портом. Клиент на экране «Присоединиться» слушает broadcast и показывает список найденных комнат — игрок просто тыкает в нужную.
2. **Ручной ввод IP** — запасной вариант, когда broadcast не проходит (гостевые Wi-Fi, VPN, разные подсети). Хост крупно показывает свой локальный IP и порт на экране лобби, клиент вводит их руками.

## Лобби
- Главное меню: «Одиночная игра», «Создать комнату», «Присоединиться».
- При создании комнаты игрок задаёт имя (по умолчанию «Комната <имя игрока>»). Хост попадает в лобби и видит подключающихся.
- В лобби: список игроков с именами и цветами собак, IP хоста, кнопка «Старт» только у хоста.
- Каждой собаке автоматически назначается свой цвет из фиксированной палитры (4 контрастных варианта), спрайт собаки перекрашивается процедурно — художника по-прежнему нет. Над собакой в игре — имя игрока мелким шрифтом.
- Проверка версии при подключении: клиент шлёт версию игры, при несовпадении — понятная ошибка «Версии не совпадают, обновите игру», а не молчаливый разрыв.

## Геймплейные отличия от одиночки
- 2–4 собаки на карте. Овцы боятся всех собак одинаково: в радиусе нескольких собак овца бежит от суммарного вектора угроз (взвешенно по близости).
- Гавк у каждой собаки свой, кулдауны независимые.
- Условие победы то же: все овцы в вольере. Победа общая на всех.
- Таймер общий, после победы показать итоговое время всем. Опционально — сколько овец загнал каждый (по последнему касанию), но это украшение, не обязательно.
- Больше собак — проще гнать стадо, поэтому для мультиплеера увеличить число овец: базовые 300 умножать примерно на 0.7 × число игроков (тюнингуемая константа).

## Обрывы и краевые случаи
- Отвалился клиент: его собака исчезает с карты, игра продолжается. Переподключение в идущую партию — просто заново зайти в комнату, хост выдаст новую собаку.
- Отвалился хост: у клиентов сообщение «Хост отключился», возврат в меню. Миграцию хоста не делать — не тот масштаб.
- Пауза в мультиплеере отсутствует; выход одного игрока не останавливает партию.
- Хост играет без сети как обычно: его локальная симуляция и есть источник истины, свой ввод он подаёт в неё напрямую, минуя сокет.

## Что осознанно НЕ делаем
- Игру через интернет, NAT-пробитие, relay-серверы, матчмейкинг.
- Защиту от читов — играют друзья, хост-авторитарность и так закрывает рассинхрон.
- Чат — люди в одной комнате или в голосовом канале.
- Больше 4 игроков.

## Порядок работ
1. Вынести игровой ввод за интерфейс: симуляция принимает команды «собака N бежит в точку», «собака N гавкает» независимо от того, пришли они с локальной мыши или из сокета. Одиночная игра после рефакторинга работает как раньше.
2. Сериализация снапшота и его применение на клиенте с интерполяцией.
3. WebSocket сервер/клиент, лобби, подключение по IP.
4. UDP-обнаружение комнат.
5. Полировка: цвета собак, имена, обработка обрывов, проверка версий.
